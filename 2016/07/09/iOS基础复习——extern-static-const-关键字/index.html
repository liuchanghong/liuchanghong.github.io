iOS基础复习——extern,static,const 关键字


  • const
    • const 与宏的区别 http://www.jianshu.com/p/2fd58ed2cf55
      • 之前常用的字符串常量，一般是抽成宏，但是苹果不推荐我们抽成宏，推荐我们使用 const 常量。
        • 宏是预编译（编译之前处理），const 是编译阶段
        • 宏不做检查，不会报编译错误，只是替换，const 会编译检查，会报编译错误。
        • 宏的好处 : 宏能定义一些函数，方法。 const 不能。
        • 宏的坏处 : 使用大量宏，容易造成编译时间久，每次都需要重新替换。
    • const作用 ：限制类型
      • const仅仅用来修饰右边的变量（基本数据变量p，指针变量p）
      • 被const修饰的变量是只读的
      • 这两种写法是一样的：const int b = 20; int const b = 20;
  • extern
    • 用于变量的声明，告诉编译器：已经存在一个全局变量，但是不在当前的编译单元内，需要连接的时候在其他编译单元中寻找。
    • 如果在其他文件中访问一个类的全局变量，可以不用导入.h文件，通过extern去直接访问。
    • 仅适于修饰全局变量，不能去修饰其他的变量。一般情况下默认，定义的全局变量都带有 extern ,所以不过多解释。
  • static
    • 修饰全局变量
      • 在全局变量前加 static ，全局变量就被定义成为一个全局静态变量（全局变量和静态全局变量的生命周期是一样的，都是在堆中的静态区，在整个工程执行期间内一直存在）
        • 存储区：静态存储区（静态存储区在整个程序运行期间都存在）
        • 作用域：静态全局变量在本文件之外是不可见的。准确地讲作用域变为从定义之处开始到文件结尾。
        • 非静态全局变量的作用域是整个源程序的生命期（多个源文件可以共同使用）
      • 修改变量作用域为当前编译单元，变量生命周期不变
      • 避免重复定义全局变量
    • 修饰局部变量
      • 在局部变量之前加上关键字 static ，局部变量就被定义成为一个局部静态变量。
        • 存储区：由栈变为静态存储区
        • 生存期为整个源程序的生命期
        • 只能在定义该变量的函数内使用，退出该函数后，尽管该变量还继续存在，但不能使用它
        • 作用域：作用域仍为局部作用域
  • static 和 const 的联合使用
    • static 与 const 联合 : 声明一个只读的静态变量
    • 开发使用场景:在一个文件中经常使用的字符串常量，可以使用 static 与 const 组合
    • 作用：extern与const组合，只需要定义一份全局变量，多个文件共享。并且全局常量只被定义一次，分配一次内存空间。
    • 写法：.h 中定义全局变量 extern NSString const lhString; | .m 中写 NSString * const lhString=@”hello”; 在其他文件中调用时 引入 .h 头文件即可
